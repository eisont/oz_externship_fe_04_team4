# 커밋 컨벤션

## 1. 기본 규칙

- 형식: `<type>(<scope>): <subject>`
- type은 소문자 사용
- subject는 한글 사용 가능, 끝에 마침표 X

## 2. type 종류

- feat: ...
- fix: ...
- chore: ...
- refactor: ...
- style: ...
- docs: ...
- test: ...
- ci: ...
- build: ...
- perf: ...

## 3. 예시

- feat: 커뮤니티 카테고리 페이지 SSR 적용
- fix: 로그인 실패 시 에러 메시지 문구 수정
- chore: ESLint + Prettier 설정 추가
- style: Prettier 포맷 적용
- docs: API 응답 스펙 문서 추가
